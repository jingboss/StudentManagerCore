# 空闲超时 Bug 修复

## 问题现象
- 用户长时间不操作，超时警告弹窗出现
- 但倒计时不会归零，弹窗永远不会自动退出登录
- 用户点"继续会话"有时也不生效

## 根因
- 鼠标移动（`mousemove`）、键盘（`keydown`）、点击（`click`）等事件监听器在**超时警告弹窗显示时**仍会调用 `resetIdleTimer()`
- `resetIdleTimer()` 会清除当前的倒计时 `setInterval` 并重新开始
- 这导致弹窗上的倒计时被反复重置，永远到不了0

## 修复
- 在 `resetIdleTimer()` 顶部添加 `if (modalShown) return;`
- 弹窗显示时，普通交互不再重置计时器
- "继续会话"按钮有独立的重置逻辑，不受此限制

## 涉及文件
- `Views/Shared/_Layout.cshtml` - idle timer 逻辑
