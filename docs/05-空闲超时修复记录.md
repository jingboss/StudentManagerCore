# 空闲超时 Bug 修复

## 问题

超时警告弹窗出现后倒计时永不归零，弹窗不会自动退出登录，"继续会话"按钮也不生效。

## 原因

mousemove/keydown/click 事件在弹窗显示时仍调用 resetIdleTimer()，它会清除并重置倒计时。弹窗倒计时被反复重置，永远到不了0。

## 修复

- resetIdleTimer() 顶部添加 `if (modalShown) return;`
- 弹窗显示时，普通交互不再重置计时器
- "继续会话"按钮有独立重置逻辑

## 文件

- `Views/Shared/_Layout.cshtml` - idle timer 逻辑
