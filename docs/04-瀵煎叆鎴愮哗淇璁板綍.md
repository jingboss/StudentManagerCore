# 导入成绩 SaveImport 404/500 修复

## 问题现象

用户上传Excel成功（ImportPreview通过），但点击"确认导入"时出现：
1. **404 (Not Found)** 
2. **500 (Internal Server Error)** - "系统出了点小问题"

## 根因分析

### 404 原因
- `Program.cs` line 237: `app.UseStatusCodePagesWithReExecute("/Home/Error", "?statusCode={0}")` 拦截4xx/5xx状态码并重执行到 `/Home/Error`
- `HomeController` 的 Error 方法之前对 AJAX 请求没有返回 JSON，返回 HTML 导致前端解析失败
- 当 `[ValidateAntiForgeryToken]` 验证失败（返回400）或 JWT 过期时，StatusCodePages 中间件重执行到 `/Home/Error`
- 没有针对 AJAX 的处理导致最终错误信息不可读

### 500 原因
- `SaveImport` 方法没有 try-catch
- 任何异常（数据库错误、空引用等）传播到全局异常处理器
- 全局异常处理器返回 HTML（"系统出了点小问题"），但 AJAX 期望 JSON

### LINQ 翻译错误
- 查询中使用了 `GradeLevel.CurrentGradeName` 和 `GradeLevel.DisplayName` 这两个 `[NotMapped]` 属性
- EF Core 无法将它们翻译成 SQL，导致查询异常

## 修复内容

### 1. HomeController.Error 增强
- 对 AJAX 请求返回 `Json(new { success = false, message })`
- 对页面请求返回 View

### 2. SaveImport 添加 try-catch
- 整个业务逻辑包裹在 try-catch 中
- 异常时返回 `Json(new { success = false, message = "导入失败：..." })`
- 保持原有的验证逻辑不变（满分校验、已有成绩更新等）

### 3. 查询优化
- 预加载所有 `ClassInfo` + `GradeLevel` 数据到内存
- 使用字符串字典快速查找，避免循环内反复查数据库
- 不再在 LINQ 中使用 NotMapped 属性

## 涉及文件

- `Controllers/HomeController.cs` - Error Action 增强
- `Controllers/ScoreController.cs` - SaveImport try-catch + 查询优化
- `Views/Score/Import.cshtml` - 前端 AJAX 调用（未修改）
